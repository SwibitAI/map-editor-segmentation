# Map Editor — Segmentation Tool Demo

A browser-based map editing toolkit for digitizing land parcel maps from CAD/PDF sources.
Extracts vector line data from PDFs, connects fragmented segments into clean polygon outlines, and provides an interactive block editor for defining and subdividing land parcels.

---

## Tools

### 1. `block-editor.html` — Interactive Block & Unit Editor

The main editor. Load a map image, draw block polygons, and subdivide them into columns and rows of units.

**Features**
- Draw polygon blocks on any background map image
- Baseline-aligned grid: columns perpendicular, rows parallel to a chosen baseline edge
- Selection levels: **Block / Column / Row / Unit**
- Drag dividers to resize — proportional redistribution with lock support
- Per-column independent row counts (e.g. B59-type blocks where columns have different row depths)
- Column mode: double-click adds/removes horizontal row dividers per column
- Row mode: double-click adds/removes vertical sub-dividers within a row band
- Undo/redo, vertex editing, block merge
- GeoJSON export with block + unit features and unit numbers

**Usage**
Open in any modern browser (no server needed for basic use):
```
open block-editor.html
```
Or serve locally:
```
python3 -m http.server 8080
# then open http://localhost:8080/block-editor.html
```

### 2. `line-connect.html` — PDF Line Segmentation Viewer

Loads extracted vector line data and connects fragmented segments into polylines and closed polygon outlines using graph tracing.

**Algorithm**
1. **Snap** endpoints to a configurable grid (closes micro-gaps from CAD export)
2. **Deduplicate** segments with identical snapped endpoints
3. **Build adjacency graph** — nodes are snapped endpoints, edges are segments
4. **Trace chains** — walk through degree-2 nodes, stop at junctions
5. **Detect closed polygons** — chains where start ≈ end
6. **Collinear merge** (optional) — merges overlapping segments on the same infinite line

**Controls**
| Control | Effect |
|---|---|
| Snap slider (0.1–10 pt) | Grid cell size for endpoint snapping; larger = bridges bigger gaps |
| Min length filter | Discards segments shorter than threshold (removes noise) |
| Collinear merge | Merges overlapping collinear segments into one |
| Layer toggles | Show/hide any PDF layer independently |
| Mode: Raw | All segments color-coded by layer |
| Mode: Connected | Deduplicated segments + traced polylines + polygon fills |
| Mode: Polygons | Only closed polygons |
| Export GeoJSON | Exports all detected polygons and polylines |

**Typical results** (teebat-alnajaf.pdf, snap=1pt)

| Step | Count |
|---|---|
| Raw segments | 98,215 |
| After snap + dedup | 29,203 |
| Traced polylines | 22,297 |
| Closed polygons | 337 |

**Usage** — requires `lines_data.json` generated by `pdf_lines.py`:
```
python pdf_lines.py input.pdf output/lines_data.json
python3 -m http.server 8080
# open http://localhost:8080/line-connect.html
# file auto-loads from output/lines_data.json
```

### 3. `pdf_lines.py` — PDF Vector Line Extractor

Extracts all stroke paths from a CAD/map PDF as line segments and saves them as JSON for `line-connect.html`.

**Requirements**
```
pip install PyMuPDF
```

**Usage**
```bash
# Extract all layers
python pdf_lines.py input.pdf output/lines_data.json

# Extract specific layers only
python pdf_lines.py input.pdf output/lines_data.json --layers "Home (1),0_UNIT-1"

# List all layers in the PDF (to identify which ones to use)
python pdf_lines.py input.pdf
```

**Output JSON format**
```json
{
  "page": { "width": 1191.0, "height": 842.0 },
  "layers": ["0", "Home (1)", "0_UNIT-1", ...],
  "segments": [
    { "a": [x1, y1], "b": [x2, y2], "layer": "Home (1)", "color": "#000000", "width": 0.0 }
  ],
  "texts": [
    { "text": "B12", "x": 543.2, "y": 210.5, "size": 8.0, "color": "#000000" }
  ]
}
```

### 4. `map_extract.py` — PDF to Image Extractor

Renders a CAD/map PDF to PNG with text removed, and optionally produces a separate transparent text-only overlay.

**Requirements**
```
pip install PyMuPDF Pillow numpy
sudo apt install poppler-utils   # or: sudo dnf install poppler-utils
```

**Usage**
```bash
# Save both clean base image and text overlay
python map_extract.py input.pdf --both ./output/ --dpi 150

# Widen lines for better visibility (multiply all line widths by 2×)
python map_extract.py input.pdf --no-text clean.png --dpi 150 --line-scale 2.0

# Quick preview at 72 DPI
python map_extract.py input.pdf --no-text preview.png --dpi 72

# List PDF layers
python map_extract.py input.pdf --list-layers
```

**Options**
| Flag | Default | Description |
|---|---|---|
| `--dpi N` | 150 | Render resolution. 72 = 1pt:1px, 150 = best quality/size balance |
| `--line-scale F` | 1.0 | Multiply all line widths before rendering (2.0 doubles thickness) |
| `--both DIR` | — | Save both `*_no_text.png` and `*_text_only.png` to directory |
| `--no-text PATH` | — | Save clean image only |
| `--text-only PATH` | — | Save transparent text overlay only |
| `--list-layers` | — | Print all OCG layers and exit |

---

### 5. `overlay.html` — Base + Text Layer Viewer

Leaflet-based viewer that overlays the text-only PNG transparently on top of the clean base image. Useful for inspecting text label positions before digitizing.

**Usage**
```
python3 -m http.server 8080
# open http://localhost:8080/overlay.html
```
Use the **Base** and **Text** buttons in the toolbar to swap images at runtime.

## Typical Workflow

```
input.pdf
    │
    ├─ map_extract.py --dpi 150 --line-scale 2
    │       └─→ map_no_text.png          ← background image for block-editor
    │       └─→ map_text_only.png        ← optional text overlay for overlay.html
    │
    └─ pdf_lines.py
            └─→ output/lines_data.json   ← feed into line-connect.html
                    │
                    └─ line-connect.html
                            │  (adjust snap, filter to block-boundary layers)
                            └─→ lines_export.geojson   ← detected block polygons
```

Then in `block-editor.html`:
1. Load `map_no_text.png` as background
2. Use detected polygon outlines from `line-connect.html` as reference to draw blocks
3. Set baseline + grid divisions per block
4. Export final `blocks.geojson`

---

## Coordinate System

- PDF coordinates: points (pt), origin top-left, Y grows downward
- At 150 DPI: `image_px = pdf_pt × (150/72) = pdf_pt × 2.0833`
- Block editor world coordinates = image pixel coordinates
- `line-connect.html` and `block-editor.html` share the same Y-down convention

---

## Browser Compatibility

All HTML tools use only Canvas 2D API and vanilla JS — no build step, no dependencies beyond Leaflet (loaded from CDN in `overlay.html`). Works in Chrome, Firefox, Edge, Safari.
